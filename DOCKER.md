# Запуск через Docker — подробная инструкция

Проект: `D:\electronic-library`

---

## Где что делать (кратко)

| Что делать | Где |
|------------|-----|
| Открыть проект, редактировать `.env` | **PyCharm** (окно с файлами слева) |
| Вводить команды `docker`, `py init_db.py` и т.д. | **Терминал внизу PyCharm** (или отдельное окно PowerShell) |
| Запустить MySQL в Docker | Команда в **терминале** (Docker Desktop должен быть открыт) |
| Запустить сайт | **Терминал** (`py run.py`) или кнопка **Run** в PyCharm |
| Смотреть сайт | **Браузер** (Chrome, Edge и т.д.) |
| Запустить Docker | **Иконка на рабочем столе** Docker Desktop |

---

## ЧАСТЬ 1. Один раз: установка

### 1.1. Docker Desktop

1. Скачайте: https://www.docker.com/products/docker-desktop/
2. Установите (Next → Next).
3. После установки перезагрузите компьютер, если попросит.

### 1.2. Python и библиотеки проекта

**Где:** терминал в PyCharm (см. ниже, как его открыть).

Если PyCharm ещё не открыт:
1. Запустите **PyCharm**
2. **File → Open**
3. Выберите папку `D:\electronic-library` → **OK**

**Как открыть терминал в PyCharm:**
- Внизу окна вкладка **Terminal**
- Если не видно: меню **View → Tool Windows → Terminal**

В терминале внизу по очереди введите (после каждой строки — Enter):

```powershell
cd D:\electronic-library
```

```powershell
py -m venv venv
```

```powershell
venv\Scripts\activate
```

После последней команды слева в строке появится `(venv)` — это нормально.

```powershell
pip install -r requirements.txt
```

Подождите, пока всё установится.

### 1.3. Интерпретатор в PyCharm

**Где:** настройки PyCharm (не терминал).

1. **File → Settings** (или Ctrl+Alt+S)
2. Слева: **Project: electronic-library → Python Interpreter**
3. Справа вверху иконка **шестерёнки** → **Add → Add Local Interpreter**
4. **Existing** → три точки `...`
5. Укажите файл: `D:\electronic-library\venv\Scripts\python.exe`
6. **OK → OK**

### 1.4. Файл `.env`

**Где:** редактор PyCharm (центральная часть), дерево файлов слева.

1. Слева в списке файлов найдите `.env.example`
2. Правый клик → **Copy**
3. Правый клик по корню проекта **electronic-library** → **Paste**
4. Правый клик по копии → **Rename** → имя: `.env`

Откройте `.env` двойным кликом. Должно быть:

```
MYSQL_HOST=localhost
MYSQL_USER=root
MYSQL_PASSWORD=root
MYSQL_DATABASE=electronic_library
SECRET_KEY=change-me-in-production
```

Сохраните: **Ctrl+S**.

> Пароль `root` совпадает с `docker-compose.yml`. Менять не нужно.

## ЧАСТЬ 2. Каждый раз перед работой

### 2.1. Запустить Docker Desktop

**Где:** рабочий стол Windows, **не** PyCharm.

1. Двойной клик по **Docker Desktop**
2. Подождите 1–2 минуты
3. Внизу окна Docker должно быть: **Engine running**

Пока не running — команды `docker` в терминале не сработают.

---

## ЧАСТЬ 3. Первый запуск проекта (один раз)

Все команды ниже — **в терминале PyCharm** (вкладка Terminal внизу).

Если терминал закрыт: **View → Tool Windows → Terminal**.

### Шаг 1. Перейти в папку проекта

```powershell
cd D:\electronic-library
```

### Шаг 2. Активировать venv

```powershell
venv\Scripts\activate
```

Должно быть `(venv)` в начале строки.

### Шаг 3. Запустить MySQL в Docker

```powershell
docker compose up -d
```

Первый раз скачает образ MySQL — может занять несколько минут.

Проверка (в том же терминале):

```powershell
docker compose ps
```

Ждите, пока в колонке STATUS будет **Up (healthy)**. Может занять до минуты.

### Шаг 4. Создать базу данных

```powershell
py init_db.py
```

**Должно вывести:**
```
База данных создана.
Логины: admin, moderator, user1 | пароль: password123
```

Скрипт также создаёт тестовые обложки книг.

## ЧАСТЬ 4. Запуск сайта

### Способ А — через терминал PyCharm

В том же терминале (с `(venv)`):

```powershell
py run.py
```

**Не закрывайте** это окно терминала — пока оно работает, сайт доступен.

Должно появиться:
```
Running on http://127.0.0.1:5000
```

### Способ Б — через кнопку Run в PyCharm

**Где:** верхняя панель PyCharm, справа.

1. В выпадающем списке выберите **run.py**
2. Нажмите зелёный треугольник **Run**

Внизу откроется вкладка **Run** с логом сервера.

---

## ЧАСТЬ 5. Открыть сайт в браузере

**Где:** браузер (Chrome, Edge, Firefox) — **не** PyCharm.

В адресной строке введите:

```
http://127.0.0.1:5000
```

Нажмите Enter.

Должна открыться главная со списком книг.

---

## ЧАСТЬ 6. Проверка (тестовые логины)

**Где:** браузер, сайт http://127.0.0.1:5000

1. Нажмите **Войти** (правый верхний угол)
2. Введите логин и пароль из таблицы
3. Нажмите **Войти**

| Логин | Пароль | Что проверить |
|-------|--------|---------------|
| admin | password123 | Добавить книгу, удалить книгу |
| moderator | password123 | Модерация рецензий, редактирование |
| user1 | password123 | Написать рецензию, «Мои рецензии» |

## Повседневный запуск (уже после первой настройки)

Порядок каждый день:

1. **Рабочий стол** → запустить **Docker Desktop** → дождаться **Engine running**
2. **PyCharm** → вкладка **Terminal** внизу
3. Ввести команды:

```powershell
cd D:\electronic-library
venv\Scripts\activate
docker compose up -d
py run.py
```

4. **Браузер** → `http://127.0.0.1:5000`

`init_db.py` **больше не нужен**, если базу не удаляли.

## Ошибки

| Что видите | Что сделать |
|------------|-------------|
| `dockerDesktopLinuxEngine` | Запустить Docker Desktop с рабочего стола |
| `Can't connect to MySQL` | `docker compose ps` → дождаться healthy; проверить `.env` |
| Команда `py` не найдена | Попробовать `python` вместо `py` |
| Книг нет на сайте | В терминале: `py init_db.py` |
| Нет картинок | В терминале: `py init_db.py` |

---

## Альтернатива: PowerShell вместо терминала PyCharm

Можно не использовать PyCharm для команд:

1. Win → напечатать **PowerShell** → Enter
2. Те же команды, что в разделе «ЧАСТЬ 3»

PyCharm нужен для удобного редактирования кода и кнопки Run; команды `docker` и `py` работают одинаково в PowerShell и в терминале PyCharm.
